📥 Guide complet : Télécharger et lancer le projet

Suivez ces étapes pour télécharger, configurer et lancer l'application Coach v1 dans un émulateur Android.

📋 Prérequis :
  • Visual Studio 2022 (pas Visual Studio Code) avec le workload .NET MAUI installé
  • Android SDK installé (via Visual Studio Installer ou Android Studio)
  • Android Emulator configuré avec au moins un AVD (Android Virtual Device)
  • Windows 10/11 (pour le développement Android)

Étape 1 : Télécharger le projet

  1. Téléchargez le projet Coach v1 depuis le dépôt GitHub ou le dossier local
  2. Extrayez l'archive dans un dossier de votre choix (ex: C:\Projects\Coach V1)
  3. Vérifiez que le dossier contient le fichier MauiAppCoach.slnx ou MauiAppCoach.sln

Étape 2 : Ouvrir le projet dans Visual Studio

  1. Lancez Visual Studio 2022 (pas Visual Studio Code)
  2. Dans Visual Studio, cliquez sur "Ouvrir un projet ou une solution"
  3. Naviguez vers le dossier du projet et sélectionnez le fichier MauiAppCoach.slnx ou MauiAppCoach.sln
  4. Attendez que Visual Studio charge le projet et restaure les packages NuGet (cela peut prendre quelques minutes la première fois)
⚠️ Important : Assurez-vous d'utiliser Visual Studio 2022 et non Visual Studio Code. Visual Studio Code ne peut pas lancer les émulateurs Android directement.

Étape 3 : Configurer l'émulateur Android

📍 Où trouver la liste des appareils dans Visual Studio :

Dans la barre d'outils principale, en haut de Visual Studio, à côté du bouton vert "Démarrer" (▶)

  1. Dans Visual Studio, regardez la barre d'outils en haut (à côté du bouton "Démarrer" vert)
  2. Vous verrez une liste déroulante qui affiche probablement "Aucun appareil" ou un appareil par défaut
  3. Cliquez sur cette liste déroulante pour voir les options disponibles
  4. Si aucun émulateur n'apparaît, cliquez sur "Gérer les appareils..." ou "Device Manager"
  5. Dans la fenêtre qui s'ouvre, vous pouvez :
    • Voir les émulateurs Android déjà configurés
    • Créer un nouvel émulateur en cliquant sur "Créer" ou "Create"
    • Sélectionner un appareil (ex: Pixel 5, Pixel 6)
    • Choisir une version Android (ex: API 33 - Android 13, API 31 - Android 12)
  6. Une fois l'émulateur créé, fermez la fenêtre de gestion des appareils
💡 Astuce : La liste déroulante des appareils se trouve dans la barre d'outils principale, généralement entre le bouton "Démarrer" et les configurations de build (Debug/Release).

Étape 4 : Sélectionner l'émulateur Android

📱 Sélection de l'émulateur :

Dans la liste déroulante, choisissez un émulateur Android (ex: Pixel_5_API_33, Pixel_6_API_31, etc.)

  1. Retournez dans Visual Studio, dans la barre d'outils principale
  2. Dans la liste déroulante des appareils (à côté du bouton "Démarrer"), sélectionnez votre émulateur Android
  3. Vous devriez voir quelque chose comme :
    • "Pixel_5_API_33"
    • "Android_Emulator_Pixel_6"
    • Ou un autre nom d'émulateur que vous avez configuré
  4. Si l'émulateur n'est pas encore lancé, Visual Studio le lancera automatiquement lors du déploiement
✅ Confirmation : Une fois l'émulateur sélectionné, son nom devrait apparaître dans la liste déroulante, indiquant qu'il est prêt à être utilisé.

Étape 5 : Lancer l'application

▶️ Bouton Démarrer :

Cliquez sur le bouton vert "Démarrer" (▶) dans la barre d'outils ou appuyez sur F5

  1. Une fois l'émulateur sélectionné dans la liste déroulante, vous êtes prêt à lancer l'application
  2. Cliquez sur le bouton "Démarrer" (flèche verte ▶) dans la barre d'outils, ou appuyez sur F5
  3. Visual Studio va :
    • ✅ Compiler le projet .NET MAUI
    • 🚀 Lancer l'émulateur Android (s'il n'est pas déjà lancé - cela peut prendre 1-2 minutes la première fois)
    • 📦 Déployer l'application sur l'émulateur
    • 🎯 Lancer automatiquement l'application Coach v1
  4. L'application devrait maintenant s'afficher dans l'émulateur Android !
⏱️ Temps d'attente : Le premier lancement peut prendre 2-3 minutes. Les lancements suivants seront beaucoup plus rapides (30 secondes environ).
💡 Astuce : Le premier lancement de l'émulateur peut prendre plusieurs minutes. Les lancements suivants seront beaucoup plus rapides car l'émulateur reste en mémoire.

Méthode alternative : Lancer l'émulateur manuellement (optionnel)

Si vous préférez lancer l'émulateur avant d'ouvrir Visual Studio, vous pouvez utiliser l'une de ces méthodes :

Via Android Studio :

  1. Ouvrir Android Studio
  2. Aller dans Tools > Device Manager
  3. Cliquer sur le bouton Play (▶) à côté d'un appareil virtuel
  4. Attendre que l'émulateur démarre complètement
  5. Ensuite, dans Visual Studio, sélectionnez cet émulateur dans la liste déroulante

Via ligne de commande (PowerShell) :

# Lister les AVD disponibles $env:LOCALAPPDATA\Android\Sdk\emulator\emulator.exe -list-avds # Lancer un émulateur spécifique (remplacez "Pixel_5_API_33" par votre AVD) $env:LOCALAPPDATA\Android\Sdk\emulator\emulator.exe -avd Pixel_5_API_33

Script automatique (Windows) - Option rapide

Exécutez directement le script PowerShell pour lancer l'émulateur automatiquement :

Lancer l'émulateur Android

Note : Cliquez-droit sur le fichier téléchargé > "Exécuter avec PowerShell" ou double-cliquez si PowerShell est configuré par défaut.

# Commandes PowerShell pour lancer l'émulateur Android # Remplacez "Pixel_5_API_33" par le nom de votre AVD # Trouver le chemin de l'émulateur $emulatorPath = "$env:LOCALAPPDATA\Android\Sdk\emulator\emulator.exe" # Lister les AVD disponibles & $emulatorPath -list-avds # Lancer un émulateur spécifique & $emulatorPath -avd Pixel_5_API_33

Pour une meilleure expérience, utilisez un de ces appareils virtuels :

  • Pixel 5 API 33 (Android 13) - Recommandé
  • Pixel 6 API 31 (Android 12)
  • Pixel 7 API 34 (Android 14)
⚠️ Note : Le premier lancement de l'émulateur peut prendre plusieurs minutes. Les lancements suivants seront plus rapides.

🔧 Configuration de l'émulateur

Si vous n'avez pas d'émulateur configuré :

  1. Ouvrir Android Studio
  2. Aller dans Tools > Device Manager
  3. Cliquer sur Create Device
  4. Sélectionner un appareil (ex: Pixel 5)
  5. Télécharger une image système (ex: API 33 - Android 13)
  6. Finaliser la création de l'AVD

🔄 Résumé des étapes principales

Pour référence rapide, voici les étapes essentielles :

  1. Télécharger le projet Coach v1
  2. Ouvrir Visual Studio 2022 (pas Visual Studio Code)
  3. Ouvrir le fichier MauiAppCoach.slnx ou MauiAppCoach.sln
  4. Dans la barre d'outils, cliquer sur la liste déroulante des appareils
  5. Sélectionner un émulateur Android (ou créer-en un via "Gérer les appareils...")
  6. Cliquer sur "Démarrer" ou appuyer sur F5
  7. Attendre que l'émulateur se lance et que l'application se déploie
⚠️ Problèmes courants :
  • Pas d'émulateur dans la liste ? → Cliquez sur "Gérer les appareils..." et créez-en un
  • L'émulateur ne démarre pas ? → Vérifiez que l'Android SDK est bien installé
  • Erreur de compilation ? → Vérifiez que le workload .NET MAUI est installé dans Visual Studio Installer
  • Visual Studio Code au lieu de Visual Studio ? → Vous devez utiliser Visual Studio 2022, pas VS Code

Commandes alternatives en ligne de commande (avancé)

Si vous préférez utiliser la ligne de commande :

# Naviguer vers le dossier du projet cd "Chemin/Vers/Coach V1" # Restaurer les packages NuGet dotnet restore # Compiler le projet dotnet build # Lancer sur Android (nécessite un émulateur déjà lancé) dotnet build -t:Run -f net8.0-android
